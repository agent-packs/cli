Why Agent Packs

Know what your agents run.
Prove it hasn't changed.

Skills and plugins are code your agents act on. Agent Packs manages them like a software supply chain: recorded provenance, pinned revisions, drift detection, and a CI gate.

CI
One-Command CI Gate
Verify pins, managed-file drift, and a trust policy for every installed pack in one command with a nonzero exit on failure — made for pull-request checks.
agent-packs check --policy ci
PN
Pinned Provenance
Resolve moving refs to concrete commits and content checksums in the lockfile, then fail fast when a live source no longer matches its recorded pin.
agent-packs pin frontend-engineer --check
DF
Drift Detection
Catch hand-edits to installed skills, managed files, memory blocks, and merged settings keys since install — before upgrading or rolling back.
agent-packs status
MG
Reversible Memory & Settings
Merge CLAUDE.md/AGENTS.md fragments and settings keys add-only and user-wins, tracked in receipts and cleanly retracted on uninstall.
agent-packs install my-pack --only memory --mode copy
RB
Receipts & Rollback
Every install writes a receipt and lockfile, so upgrades, managed descriptors, and merged fragments can be audited, traced, and reverted.
agent-packs rollback backend-engineer
AI
One Tool, All Agents
Manage skills, plugins, tools, MCP servers, commands, hooks, memory, and settings across Claude Code, Codex, Cursor, Gemini CLI, and more from a single CLI.
agent-packs install backend-engineer --agent claude

How it works

Packs: one bundle,
every capability

A Pack is a named collection of capabilities — skills, plugins, managed tools, MCP servers, commands, hooks, prompts, templates, memory, and settings — tuned for a role like backend engineer, ML engineer, or SRE. One install command resolves each capability to the right behavior: record, copy, merge, or gated native setup.

Getting Started

Up and running in minutes

1

Install the CLI

Install via Homebrew, curl, Docker, or Go.

# Homebrew (recommended)
brew install agent-packs/tap/agent-packs

# curl (macOS / Linux)
curl -fsSL https://raw.githubusercontent.com/agent-packs/cli/main/install.sh | sh

# Docker
docker run --rm -it ghcr.io/agent-packs/cli:latest
2

Search the registry

Find packs by role, language, or tag.

agent-packs search backend
# → backend-engineer, rust-engineer …
3

Install into your agent

Pick a target agent and install. The default reference mode records sources; --mode copy writes managed files and merges supported settings.

agent-packs install backend-engineer --agent claude --mode copy
agent-packs install sre-engineer --agent codex --only tools --mode copy
# Receipts track every managed capability
4

Keep up to date

Update all installed packs, or just one. Diff before you upgrade.

agent-packs outdated
agent-packs diff backend-engineer
agent-packs upgrade backend-engineer
5

Pin it and gate CI

Record commit and checksum pins in the lockfile, commit it, and let one command verify pins, drift, and policy on every pull request.

agent-packs pin backend-engineer
agent-packs check --policy ci
# exits nonzero when anything drifted
